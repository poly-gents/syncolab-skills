# Syncolab Skills

First-party [Agent Skills](https://agentskills.io) for Syncolab: reusable operational playbooks under `skills/<skill-id>/`.

This repository is a **public skill library** (content only). It ships skill packages, a committed [`manifest.json`](manifest.json) index, and authoring docs. It does not include npm packages or maintainer scripts—validation and manifest generation run in [`poly-gents/syncolab-skills-tooling`](https://github.com/poly-gents/syncolab-skills-tooling). Pull requests are checked by a thin GitHub Actions workflow that delegates to that tooling repo.

## Install locally (Cursor, Claude Code, etc.)

Use the open [skills CLI](https://github.com/vercel-labs/skills):

```bash
npx skills add poly-gents/syncolab-skills
npx skills add poly-gents/syncolab-skills --skill syncolab-living-docs
npx skills add poly-gents/syncolab-skills --list
```

Install scope and agent targets are described on [skills.sh](https://skills.sh).

## Repository layout

| Path | Purpose |
|------|---------|
| [`skills/`](skills/) | One folder per skill (`SKILL.md` + `meta.yaml`) |
| [`skills/_template/`](skills/_template/) | Scaffold for new skills |
| [`manifest.json`](manifest.json) | Machine-readable index and integrity checksums (generated by tooling; commit when skills change) |

## What is a skill?

Each skill directory contains:

- **`SKILL.md`** — instructions (YAML frontmatter + markdown body; [agentskills.io](https://agentskills.io) compatible).
- **`meta.yaml`** — Syncolab routing metadata (capability, triggers, tags, relationships).

Optional per skill: `references/`, `assets/`, `scripts/` (supporting material for agents; not executed by platform import).

## Contributing

1. Fork and clone this repository.
2. Clone [syncolab-skills-tooling](https://github.com/poly-gents/syncolab-skills-tooling) alongside it (see the tooling README for setup).
3. Copy [`skills/_template/`](skills/_template/) to `skills/<skill-id>/`.
4. Follow [`skills/skill.instruction.md`](skills/skill.instruction.md) and [`skills/meta.instructions.md`](skills/meta.instructions.md).
5. Validate with tooling (`npm run validate`, `python scripts/validate_skill.py <skill-id>`).
6. Run `npm run manifest:write` in tooling and **commit the updated `manifest.json`** in this repo.
7. Open a pull request to `main`. CI must pass.

For the full publish workflow, see the **publish-skill** skill under `skills/publish-skill/`.

## Syncolab cloud

If your organization uses Syncolab, administrators import skills from this repository into the Agent Studio catalog. That path is configured and operated on the platform side—not via `npx` or files in this repo.
