# Syncolab Skills

First-party [Agent Skills](https://agentskills.io) for Syncolab digital employees: reusable operational playbooks under `skills/<skill-id>/`.

This repository is a **public skill library** (content only): skill packages, a committed `manifest.json` index, and documentation. It does not ship npm packages, install scripts, or CI tooling.

| Path | Purpose |
|------|---------|
| [`skills/`](skills/) | One folder per skill (`SKILL.md` + `meta.yaml`) |
| [`skills/_template/`](skills/_template/) | Scaffold for new skills |
| [`manifest.json`](manifest.json) | Machine-readable index + integrity (generated by [syncolab-skills-tooling](https://github.com/poly-gents/syncolab-skills-tooling)) |

**Maintainer tooling** (validate, manifest generation, CI) lives in [`poly-gents/syncolab-skills-tooling`](https://github.com/poly-gents/syncolab-skills-tooling) or `managegents/syncolab-skills-tooling` in the PolyGents monorepo.

## Install locally (Cursor, Claude Code, etc.)

Use the open [skills CLI](https://github.com/vercel-labs/skills):

```bash
npx skills add poly-gents/syncolab-skills
npx skills add poly-gents/syncolab-skills --skill syncolab-living-docs
npx skills add poly-gents/syncolab-skills --list
```

See [skills.sh](https://skills.sh) for install scope and agent targets.

## Platform import (Syncolab Agent Studio)

The Managents backend shallow-clones this repo, verifies `manifest.json`, and upserts `agent_skills`. The server does **not** run `npx` or provider install scripts.

Provider config: `managegents/master-backend/.../skill-registry/registry.json` (`syncolab` provider).

## Contributing a skill

1. Clone this repo and [syncolab-skills-tooling](https://github.com/poly-gents/syncolab-skills-tooling) side by side (or use the PolyGents monorepo).
2. Copy [`skills/_template/`](skills/_template/) to `skills/<skill-id>/`.
3. Follow [`skills/skill.instruction.md`](skills/skill.instruction.md) and [`skills/meta.instructions.md`](skills/meta.instructions.md).
4. Validate and refresh the manifest using **tooling** (see tooling README).
5. Open a PR to `main` (see **publish-skill** skill).

## What is a skill?

Each skill directory contains:

- **`SKILL.md`** — instructions (YAML frontmatter + markdown body; agentskills.io compatible).
- **`meta.yaml`** — Syncolab routing metadata (capability, triggers, tags, relationships).

Optional per skill: `references/`, `assets/`, `scripts/` (supporting material; platform import does not execute them).
