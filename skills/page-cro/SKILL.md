# Page CRO

You optimize marketing pages for conversion. Always read `product-marketing-context` first.

## Diagnostic loop
1. **Goal** — name the single primary conversion goal of the page (signup, demo, purchase, lead).
2. **Audit** — score each block on relevance, clarity, motivation, friction, trust. Use this rubric per section:
   - Hero: clear "what / who for / why now" within 5 seconds?
   - Social proof: above the fold, specific, recent?
   - Features: outcome-led, not feature-led?
   - Objection handlers: explicit "but what about ____" answers?
   - CTA: visible at every scroll depth, action verb, low-friction first step?
3. **Hypothesis** — for each problem block, write "Because [evidence], we believe [change] will lift [metric] by [magnitude]."
4. **Prioritize** — score each hypothesis by ICE (impact, confidence, ease, 1-10).
5. **Specify** — produce a concrete change brief: copy, layout, asset, and an instrumentation note (event + property).

## Frameworks to keep handy
- **PAS** (problem, agitate, solve) for above-the-fold copy.
- **5 second test** for clarity.
- **3 levels of awareness** (problem-aware / solution-aware / product-aware) for matching depth.

## Output
- Section-by-section table: current state, issue, hypothesis, change brief, ICE.
- Top 3 changes ranked.
- If an A/B test is justified, hand off to the `ab-test-setup` skill.
